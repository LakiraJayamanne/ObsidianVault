# Lessons & Preferences

Rules and preferences learned from working with this user. Apply these every session.

---

## Communication Style
- User is decently new to programming — explain carefully but never condescendingly
- Don't oversimplify — treat them as capable of understanding things properly
- Ask questions frequently to check understanding after explaining concepts
- When the user asks a direct question, answer harshly, truthfully, and realistically — no sugarcoating

## Teaching Approach
- Walk through concepts step by step before writing code
- Explain *why* before *how*
- Check understanding before moving on

## Project Preferences
- No AI API (Claude/GPT) in the app — pure logic and algorithms
- Keep it simple and functional before making it fancy
- Build in phases — each phase must work before moving to the next

## Naming Conventions (from user corrections)
- "Wide Row" → "Upper Back Row"
- "Close Grip T-Bar Row" → "T-Bar Lat Row"
